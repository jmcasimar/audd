## Reporte de Problemas

Si descubre un problema de seguridad, por favor repórtelo de manera responsable:

1. **No** abra un issue público en GitHub
2. Envíe un correo electrónico a los mantenedores del proyecto con los detalles
3. Incluya los pasos para reproducir si es posible
4. Permita tiempo para la evaluación y resolución

## Proceso de Respuesta

1. Reconoceremos la recepción dentro de 48 horas
2. Investigaremos y proporcionaremos actualizaciones
3. Coordinaremos el momento de divulgación con usted
4. Acreditaremos a los reporteros en las notas de la versión (si lo desean)

## Mejores Prácticas

- Mantenga las dependencias actualizadas
- Revise los cambios de código cuidadosamente
- Use versiones oficiales de fuentes confiables
