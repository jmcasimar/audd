# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting Issues

If you discover a security issue, please report it responsibly:

1. **Do not** open a public GitHub issue
2. Email project maintainers with details
3. Include steps to reproduce if possible
4. Allow time for assessment and resolution

## Response Process

1. We will acknowledge receipt within 48 hours
2. We will investigate and provide updates
3. We will coordinate disclosure timing with you
4. We will credit reporters in release notes (if desired)

## Best Practices

- Keep dependencies updated
- Review code changes carefully
- Use official releases from trusted sources
